C++如何转换调用

经过第一个示例，纯std::move的操作比string::swap()函数慢好几倍，能想到C++之所以会调用Specializations的函数，是因为性能的原因。比如swap std::vector时，只需要交换size, capacity 以及数据的指针，肯定比std::move的操作快。

两个一模一样的specialization函数能正常运行，因为这两个specialization绑定到了两个不同模版的函数中，这两个函数是重载的关系。

C++采用函数重载方式，编译器会将这些重载的函数视为同等级的，会利用函数重载的机制在这些函数中寻找一个最合适的，这更符合编程者的直观感受。specialization的方式可能会带来surprise。

Overload when you can, specialize when you need to.

总结

  1. std::swap并不是直接使用std::move交换二者的值，为了程序性能，会调用string/vector之类的swap函数。利用函数重载选择适合的函数。
  2. template function overloading和template specialization不是同等级关系，C++会先寻找合适的overloading function，再找对应的specialization。
